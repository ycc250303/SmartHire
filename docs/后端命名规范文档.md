# SmartHire 后端命名规范文档

本文档定义了 SmartHire 后端项目的命名规范，包括包名、类名、方法名、变量名、数据库表名等各个层面的命名规则，旨在保持代码风格统一，提高代码可读性和可维护性。

参考：[Java命名规范（二哥）](https://javabetter.cn/basic-extra-meal/java-naming.html)

| 类型         | 命名规则       | 示例                                   |
| ------------ | -------------- | -------------------------------------- |
| 包名         | 小驼峰         | com.SmartHire.seekerService.controller |
| 类名         | 大驼峰         | JobSeekerController                    |
| 接口         | 大驼峰         | JobSeekerService                       |
| 实现类       | 大驼峰 + Impl  | JobSeekerServiceImpl                   |
| 方法名       | 小驼峰         | getById, sendVerificationCode          |
| 变量名       | 小驼峰         | userId, realName                       |
| 常量         | 全大写，下划线 | MAX_RETRY_COUNT                        |
| 表名         | 小写，下划线   | job_seeker                             |
| 数据库字段名 | 小写，下划线   | user_id, created_at                    |
| URL 路径     | 小写，连字符   | /job-seeker, /user-auth                |
| 配置文件     | 小写，连字符   | application-seeker-service.yml         |
